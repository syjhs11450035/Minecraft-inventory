✨ 特色

不只是仓库表，是真的会开箱的机器人

它不是把你的存档读出来 —— 它会真的上线、走路、开箱、把内容写回资料库。 所以多人服、原版反作弊、跨次元统统支援。

🤖

真实 mineflayer 机器人

不是 mock，实际 TCP 连线 Minecraft 伺服器，会上线、走路、开容器、回应聊天指令。

📦

潜影盒自动展开

透过 NBT 解析，潜影盒被视为「1 个箱子」，但内容物会全部被算入总览。

🧮

箱 - 组 - 个 换算

1 组 = 物品最大堆叠（石头 64、剑 1…），1 箱 = 27 组，由 minecraft-data 自动取值。

🗺️

区域管理

按主仓库、末影箱区、工业区分类。每个区域可设顶点座标与上层区域，方便扫描时挑选。

💾

PostgreSQL 持久化

所有快照、区域设定都存进资料库，重启不丢。Drizzle ORM 管理 schema。

💬

简体中文 UI

Streamlit 写的控制台，深色侧栏 + 5 大分页 + 状态徽章 + 物品 emoji 图标。

🏗️ 架构

三层分明，单一资料源

浏览器 → Streamlit UI → Express API → mineflayer + PostgreSQL。 前端只负责展示，所有状态都由后端持有，避免不一致。

🚀 快速开始

三步上线，不用调环境

在 Replit 上 fork 一份就能跑，本机开发也只需要 Node、Python、PostgreSQL。

装相依 + 准备资料库

Node 套件用 pnpm，Python 用 uv 或 pip。资料库用环境变数 DATABASE_URL 指过去。

pnpm install && uv sync

pnpm --filter @workspace/db run push

启动后端 API（mineflayer 在这）

Express 服务，开在 8080。前端会透过 reverse proxy 走 /api/*。

pnpm --filter @workspace/api-server run dev

启动 Streamlit UI

开在 5000，浏览器进去 → 「⚙️ 设定」填 IP / 名称 → 侧栏按【🚀 连线】。

streamlit run main.py
